# 极客家居

如果你和我一样，想要打造属于自己的智能家居，又不想将家庭数据交给各个智能家居厂商，希望将数据落在家庭内部。

是时候动起手来，搭建属于自己的极客家居了～

## 搭建原则

- 注重隐私：所有数据及运算尽量存于本地，不直接暴露至互联网
- 保持简单：能统一用系统包管理的不再手动下载安装，方便升级与维护
- 最小安装：能使用即可，不再引入其他可有可无的额外功能

## 环境准备

需要准备的环境有

### 硬件列表

| 硬件功能 | 操作系统    | 硬件备注                                               | 个人实际            |
| -------- | ----------- | ------------------------------------------------------ | ------------------- |
| 网关路由 | OpenWRT     | 长期待机，低功耗，对CPU要求较高，最好对于AES指令有加速   | Conction J4125      |
| WIFI路由 | 华硕ASUS    | 长期待机，低功耗                                       | RT-AX56U            |
| 服务器   | VMWare ESXI | 虚拟化，性能高                                         | Dell PowerEdge T430 |
| 智能设备 | 支持Zigbee  | 支持Zigbee 3.0 的设备                                   | 若干设备            |

### 域名分配

需要申请一个域名，域名分配如下：  

| 域名               | 功能     | 备注                     |
| ------------------ | -------- | ------------------------ |
| ddns.example.com   | 外网入口 | DDNS，用于外部连接        |
| gate.example.com   | 路由网关 | OpenWRT主页              |
| data.example.com   | 数据存储 | NextCloud主页            |
| conn.example.com   | 连接控制 | HomeAssistant主页        |
